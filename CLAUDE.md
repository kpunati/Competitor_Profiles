## ALWAYS read first when drafting USWM-facing content

These two files are **mandatory context** for any prompt that produces material going on USWM's website, blog, emails, presentations, or anywhere USWM-branded:

- [`_context/brand_voice.md`](_context/brand_voice.md) — USWM's voice, three convictions, vocabulary, signature phrases, content rules
- [`_context/brand_what_not_to_make.md`](_context/brand_what_not_to_make.md) — anti-patterns: cliché phrases, generic structures, tonal mistakes to avoid

Open both before writing. The brand has specific conventions:

- **Three convictions** (lean on these wherever possible): "the trend is your friend" / "risk first, returns second" / "if we're not a fit, we'll tell you"
- **Gold-italic emphasis on standout phrases** — `***Real conviction***`, `***Built around your life***`
- **Specific investment vocabulary**: trend-following, rules-based, momentum, options overlay, position sizing, risk management, capital preservation, fee-only, fiduciary, active management
- **Long avoid-list**: "comprehensive", "holistic", "your trusted partner", "world-class", "best-in-class", "bespoke", "leverage" (as verb), "synergy", "our team of experts", "decades of experience", "we are passionate about", "robust", "boutique" — see `brand_what_not_to_make.md` for the full list

Skip the brand files and outputs sound like every other RIA.

## Canonical file map

| Want | Read |
|---|---|
| Quick brief on one competitor | `Competitors/<Firm>/1_Summary.md`, or the mirror at `Summaries/summary_<Firm>/1_Summary.md` |
| Deep analysis of one competitor | `Competitors/<Firm>/2_Full_Profile.md` |
| Structured fields (tags, AUM, threat, jurisdiction, segments) for one firm | `Competitors/<Firm>/metadata.json` |
| Structured fields for ALL firms in one file | `00_Market_Overview/competitor_catalog.json` |
| A competitor's blog/article content (where captured) | `Competitors/<Firm>/Knowledge_From_Source/_index.json`, then individual `*.md` files in that folder |
| A competitor's captured site pages | `Competitors/<Firm>/Source_Data/pages/*.json` (home, about, contact, services, pricing, blog index) |
| All captured pages by kind across the catalog | `00_Market_Overview/pages_by_kind.json` (e.g., every contact page, every services page) |
| Cross-cutting matrix | `00_Market_Overview/Head_to_Head_Matrix.md` |
| Our positioning context | `_context/our_thesis.md` |
| Wealth-management category background | `_context/industry_frame.md` |
| Controlled vocabulary (valid tag values, threat-level rubric, jurisdiction conventions) | `_context/taxonomy.md` |

## How to answer common question patterns

- **"What does competitor X say about Y?"** → read `Competitors/X/Knowledge_From_Source/_index.json` first to find relevant posts, then read the matching `.md` files. Fall back to `2_Full_Profile.md` if no corpus exists for that firm (see "Catalog state" auto block above for which firms have corpora).
- **"Find all firms that are <tag>"** → grep `Competitors/*/metadata.json` for the tag (consult `_context/taxonomy.md` for valid tag values — all snake_case, single canonical vocabulary).
- **"How do firms handle <page kind>?"** → read `00_Market_Overview/pages_by_kind.json` to enumerate; then read individual page JSON files.
- **"Who's most similar to USWM?"** → walk `Summaries/summary_*/1_Summary.md` (fastest mirror), filter against USWM's **durable identity** (offerings + voice + spec — see `_context/brand_voice.md` and `_context/our_thesis.md`): fee-only fiduciary · active investment management (trend-following + options overlay + rules-based + risk-first) · tax-integrated (EA + NATP credentials, real tax work) · three convictions as operating philosophy · NJ-anchored. Size, headcount, AUM are current state but not the filter — USWM is growth-ambitious, so the right comparison is **methodology and voice**, not team size.
- **Drafting in USWM voice** → start by reading `_context/brand_voice.md` + `_context/brand_what_not_to_make.md`. Then read 3-5 relevant competitor pages. Then write. Cite which competitor pages you drew from at the end.

- **Markdown** for drafts unless asked otherwise
- **Cite repo-relative paths** for any claim (e.g. *"per `Competitors/Money_Guy/1_Summary.md`…"*) — helps the reader verify

## Known data-quality issues — surface honestly

When your answer touches these, mention the caveat in the response:

- **`Advize_Wealth` blog post titles** show the firm's footer address widget ("429 S. Main Street / Rochester, MI 48307") instead of the real titles. Wix template artifact. The post **bodies are correct**; only the frontmatter `title` is wrong. Use the body's first H1, not the title field.
- **IAPD false positives** — `Streamline_Planning`, `Asset_Map`, and `Starke` are mismatched to unrelated SEC filings. Each metadata file's `regulatory_note` documents the false positive. Treat their `sec_iapd.*` fields (CRD, SEC#, AUM) as unverified.
- **~8 firms hit Cloudflare bot-blocks during site capture** — their `Source_Data/pages/*.json` may contain 403 challenge body text rather than real page content. Worst case: `Samo_Financial` (100% challenge pages). Cross-check against `Knowledge_From_Source/` (more reliable) or skip site-shape claims for those firms.
- **`Money_Guy` Knowledge corpus** points at `/resources/` (resource index), not `/articles/` — so ~half the captures are calculator/PDF-download landing pages, not articles. Read the body before treating as a blog post.
- **`Pwl_Capital` corpus** is mostly service pages — their real blog content lives at `rationalreminder.ca` + YouTube, not their main domain.
- **Catalog is a snapshot.** Each firm's `next_review_by` in metadata tells you when it was last verified. For verifiable changes (pricing, leadership, AUM), check `next_review_by` and qualify time-sensitive claims accordingly.

## What NOT to do

- **Don't modify `Competitors/*/metadata.json` directly** — fields feed `tools/synthesize.ts` and downstream auto-indexes. Suggest changes; let the maintainer apply.
- **Don't write new competitor profiles ad-hoc.** The pipeline (`tools/add.ts` → `extract.ts` → `enrich-adv.ts` → `enrich-adv-pdf.ts`) is the canonical path. If a firm should be added, name it and ask.
- **Don't invent data.** If a question can't be answered from the files, say so plainly. Don't fabricate AUM, headcounts, founding dates, or claims.
- **Don't quote competitor blog posts as USWM positions.** Their content is competitive intel; USWM's voice is in `_context/brand_voice.md`.
- **Don't write content that triggers the avoid-list** in `_context/brand_what_not_to_make.md`. Generic financial-services copy is the failure mode — every choice should be more specific, more honest, more editorially distinct than the category average.
